A Note on Source Order

There are some really fun parts of web development. HTML tends to not be one of those parts. But as the skeleton of the Web it is generally all that the non-humans see when they look at a web page.

Source order is the order of the elements in the code. Whatever you write in the HTML is the source order of the page. A bot or screen reader will access your code this way and read top-to-bottom, left-to-right.

With the advent of Flexbox and CSS Grid, we have the ability to change the visual order (what the user sees on the page) without changing the source order. It was not that way 20 years ago. Doing this, however, really messes with the ability for users to use the keyboard to navigate, causing accessibility issues. I'm not going to get into the tabindex attribute in HTML, but you can read about the importance of source order for tabbing through a web page at MDN.

Designers rarely have to think about accessibility. A good designer will understand whether their text and color contrast meets accessibility requirements or not, but many accessibility concerns fall into the laps of developers. Devs are directly responsible for how the HTML is output on the page. They are responsible for making the page easier for bots to read. They are responsible for writing semantic, legible code.

My best advice when you get a design is to reproduce the design in a way that makes sense if the content were read out loud. Take a card layout, for example. What is the hierarchy of the card content from most important to least important? In that case the order should be headline, description, photo, and then link. That is the best way to present the card in the HTML. However, visually it is more standard to present the photo first then the headline, description, and link. This is how we process information visually. In this case I would write the code so that a screen reader would present the most important information first. The photo is not as important and can be pushed down in the source code. We can always bring it back up in the visual order through CSS.

This is just a small example but shows how we need to be careful to order our HTML well so that the page can be easily understood and that it makes sense to users who are not accessing the page visually (which includes bots). Something to think about the next time you get a design and begin processing how that should be translated to code.

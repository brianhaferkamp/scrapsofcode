Creating a Web App Manifest

Last week I posted how to add a service worker to your website. The service worker is the first piece of turning your website into a progressive web app (PWA). The second piece is to have a manifest installed in the root folder of your project (the same place your service worker file is located).

What is the manifest? It is a JSON file that tells the browser about your web app. You should save the following code in a file called manifest.json and put the file in the root of your website project:

{
  "short_name": "Name",
  "name": "Longer Name Here",
  "icons": [
    {
      "src": "/images/icons-192.png",
      "type": "image/png",
      "sizes": "192x192"
    },
    {
      "src": "/images/icons-512.png",
      "type": "image/png",
      "sizes": "512x512"
    }
  ],
  "start_url": "/?source=pwa",
  "background_color": "#3367D6",
  "display": "standalone",
  "scope": "/",
  "theme_color": "#3367D6"
}

Next, you need to let the web page know where the file is and link to it. You simply add this link tag to the head of each page in your app:

<link rel="manifest" href="/manifest.json">

Now that you have the basic framework, let's walk through each piece.

The first part of the manifest are the name and short_name. You must provide one of these. If both are used then the short name is used on the home screen when the user adds your website to the home screen. It's also used in the launcher or wherever there is limited space. The longer name is used for the install prompt, if this functionality is included.

The second piece of the manifest is the icons. These are the icons that are used when the user adds your app to the home screen and for the launcher/splash page. Include a large and small icon here—one at 192px and one at 512px.

The start_url is the URL you want the user to go to when they open your app from the home screen. Often it will be the home page of your site, but maybe you have a special start screen for app users. You can append a query string to the URL so you can track how many people are engaging your site via the home screen as opposed to just typing in the web address in the browser. In our example above, the query string is ?source=pwa. You can make this whatever makes sense to you.

When your app is first launched from the home screen, there is a splash screen that is shown to the user. The splash screen has your icon and a background color. The background_color key in your manifest controls the background color of the splash screen.

There are multiple display modes that you can choose for your app. The display key controls which one you want app users to see. The options are:

  • fullscreen. The user opens the app without any browser controls. It looks and behaves like a native app. Be aware that you are full in control of the user's experience, so your app must have an appropriate and thoughtful UI.
  • standalone. Similar to fullscreen mode but the app is not opened in the browser. In this mode it is most like a native app. If you want to show the Add to Home Screen prompt, the app must be set to standalone.
  • minimal-ui. Similar to fullscreen mode but it does include some minimal browser controls for navigation. This is only supported in Chrome mobile.
  • browser. Opens your app in a standard browser.

The scope determines how links are opened in your app. Any links opened inside your app will not be opened in a new tab. You can force a new tab by using <target="_blank"> in your <a> tags. If you don't include a scope, it is implied that the scope is the directory you are serving your web app from.

The theme color sets the tool bar above the app. This is where your notifications, time, network connectivity, etc. are located. You can make that any color you wish. You also need to add this meta theme color to the head of your page:

<meta name="theme-color" content="#4285f4">

Once you've filled in the boilerplate information, you can check to see that your manifest is loaded properly by going to your website and opening the dev tools.

A Mini-Course on Creating Responsive Images for the Web

After a little break for American Independence Day, we're back at it!

One of the things everyone can improve on is working with images for the Web. It's vital to get the best quality image at the lowest possible file size. This helps keep sites fast and doesn't force the user to use up their data unnecessarily. Think about it. If you could save someone 50-90% of the data they would use and not change their experience, why wouldn't you? It's a part of user experience and will help set off your work as better than others.

I created a 2-part video series about image editing for the Web. I typically use a small image editing program (Windows only) that helps do all the little daily tasks: cropping, resizing, resampling, desaturate, save for web, etc. It's smaller than Photoshop or Lightroom, is very stable, and it's free (plus, it includes no spyware/malware). I talk about the program in video 1.

In the second video I discuss how to implement responsive images in the HTML code. This has been made very easy with the <picture> element in HTML5.

Enjoy these two videos! I'll be back on Monday with something new. There is a link in the video description of the second video to a pen on Codepen where you can see the responsive code in action.

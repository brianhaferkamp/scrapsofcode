A Brief Apology for jQuery

I would consider myself a hybrid designer/developer. Development skills are what puts money into the bank account but I am a designer at heart. I started learning JavaScript back in the early 2000s, long before it was "everywhere." Back then I understood the object oriented programming concepts but JavaScript just didn't click like HTML and CSS. So I would get frustrated and go back to design and UI.

When I started working in web development I needed to get some everyday JavaScript under my belt so I could just do the expected levels of interaction on a given website. (There is an expected level.) So I started watching some DevTips videos where Travis Neilson was showing how to work with jQuery. (Here's a good playlist to learn jQuery in 15 minutes.) Mostly it was just the basics of how to use it and what you can do with it. Being a designer-developer, Travis' videos were perfect for me. I felt that we spoke the same language. So I started working with jQuery almost exclusively (although now I know how to convert things back and forth to vanilla JS, when necessary).

For years now people have tried to get away from jQuery because it's an extra 30kb download (30kb...for shame!). I mean, seriously, the average web page is over 2mb. I don't really want to hear talk about an extra 30kb.

For me, jQuery represents a bridge to JavaScript, much like Algebra is a bridge to higher level mathematics for many. jQuery uses real words. It shortens things up so you can just use a method to do things. It makes my code more readable. Essentially, it's designer speak not developer speak.

I'm sticking with jQuery.

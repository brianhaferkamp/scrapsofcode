My Everyday Web Development Tools

As web developers, we use a lot of tools. Most people with a craft do. So here are some of my favorite tools to use.

Adobe XD

My work team has adopted Adobe XD for all of our design work. I love its simplicity and that we can create interactive mockups that can be shared on the Web. Animations, vectors, and more make this program an amazing addition. Photoshop and Illustrator work well, but they aren't necessarily made for what we do. XD hits all the right notes for a web developer.

FastStone Image Viewer

This little Windows program is an essential part of my image editing workflow. When Lightroom or Photoshop are too heavy I turn to FastStone Image Viewer to do light editing. I also use the screenshot tool all the time to grab images of whole windows or specific areas. It opens quickly and does all the little jobs perfectly.

Atom

I've been using Atom as my code editor for years. I love that it's open source, configurable, and really lightweight. It's got all the basics plus people write plugins for specific libraries and languages. It also connects very easily to Github so you can push and pull right from Atom. Slick.

My Typical Stack

About half of our projects at work are WordPress and the other half use my typical stack: PugJS, SASS, jQuery, BrowserSync, Gulp. It's a node-based workflow that I've added to over the years to get up and running quickly. It includes a material design icon font.

I call my build system, The 20-Mile Head Start, because if building a site were a marathon, this system is like being driven to mile 20 and then starting the race. It saves so much time in setup and all you need to get going is node.js.

WordPress

When I reach for a CMS it's always WordPress. The community is so large, the resources plentiful, and PHP is easy to work with. It's a no-brainer. That said, I try to steer our projects away from a CMS unless completely necessary. There is some maintenance work that must go into WordPress so building with it means maintaining it for the long haul. That's something I try to keep in mind in recommending a stack for each project.

Webflow

I've just started to get into Webflow over the past year. It's not what I thought it was--in a good way. I thought it would be a Wix-type editor, but that's not the concept. Webflow is a visual text editor. That's the best way to describe it. There aren't a lot of prebuilt components out of the box that you simply drag and drop into a site. Rather, it is a visual way to write code. If you take that thought into it I think it will make more sense. It also makes it enjoyable for web developers to use who already know how to code. This is no little sweet cupcake of a website; it is a full development program. Kudos to the Webflow team.

FYI, the ease of creating relatively sophisticated JavaScript interactions has almost made me switch to it exclusively.

That's certainly not all of the tools in my toolbox. I've left out lots of good ones (for instance, jslint.com and these material design shadows on Codepen). But these are the main tools I use on a daily basis. If they might be helpful to you, check them out and give them a try.

What are some of your favorite tools to use for web development?

Responsive Images for the Web

In this video, I delve into responsive images for the Web. In days past we were only able to serve one image for the img element. However, with HTML5, we were given the picture element. It allows us to serve multiple images with the same HTML element and to do so at predetermined viewport widths.

The video covers:

  • Art direction for images (how to crop images for different sizes)
  • Saving images for the Web (the visual quality of the image)
  • Writing the HTML so that different viewport sizes receive different images

Here is the code on Codepen if you just want to get right to the HTML or follow along with the video.

See the Pen Images by Brian Haferkamp (@brianhaferkamp) CodePen.

Why is all of this important? It is a user-centered approach. Serving one image for all screens is not ideal because small devices are usually on data networks or Wi-Fi, which are inherently slower than Ethernet connections. Small devices also have less processing power than desktop or laptop computers. This all means users experience slower loading times and use more bandwidth (i.e., data) for no particular reason.

A really cool advantage of this technique is you get to direct how users see images at certain screen sizes. If you serve one image that looks great on a large desktop it might not look as good on a mobile phone. Some of the details will be lost because the image is shrinking to a smaller overall size. Using the technique in the pen above, you can crop an image differently for different sized devices to show the user exactly what you want them to focus on in the image.

Use this really easy technique to serve appropriate images to your users.

Easy Centering with CSS Grid

Centering in CSS used to be a pretty painful situation. It still can be if you have to center absolutely positioned elements (but it really doesn't have to be). However, we now have more tools than ever to center relatively positioned elements perfectly in a responsive environment. I will show you how to center text over an image using CSS Grid Layout with a Flexbox fallback.

This HTML code is pretty typical of a hero or feature element that I work with daily:

<header class="hero">
  <div class="hero-text">
    <h1>Some Hero Text Here</h1>
    <h2>Some hero subtext goes here and is a little longer</h2>
    <button>Hero Button</button>
  </div>
</header>

There is a .hero element with a background image and text overlay. This example has a button.

We're going to use CSS Grid Layout to center these items and also provide a CSS Flexbox fallback on the off chance that a user is using an older browser that doesn't support CSS Grid Layout. At this point we're at over 90% of all browsers providing unprefixed support of Grid (according to caniuse.com). Until that gets to 100% it's wise to provide some sort of Flexbox fallback—which every major browser supports (even IE11).

I did set some base styles for our HTML above. I won't go into them here, but you can see them in the Codepen I created. Let's focus in on the Flexbox and Grid code.

Before we move on, an important concept for both Flexbox and Grid is that currently they only support parent-child relationships. Setting a parent to display: flex or display: grid only affects the child elements of that container, not the grandchildren. So in our example above I have added an extra <div> with the class hero-text. This groups our text elements together and allows us to center them as one block. It becomes the child which will be acted upon by the .hero element.

First, let's set up the fallback code. This is how we could easily center using Flexbox:

.hero {
  display: flex;
  justify-content: center;
  align-items: center;
}

This sets .hero to use Flexbox and then centers horizontally (justify-content) and vertically (align-items). Most browsers will read this code and not have any problems centering it.

With our fallback in place, we can scroll to the bottom of our CSS file and add our CSS Grid centering. This is done with a feature query, which uses the @supports query. Here is what that code looks like:

@supports (display: grid) {
  .hero {
    display: grid;
  }

  .hero-text {
    place-self: center;
  }
}

Whatever code we put into the @supports query will be read and used by browsers that support whatever feature we define. This is a way to do progressive enhancement for new CSS features. So in this code we are checking to see if the browser supports display: grid.

So we set our .hero element to display: grid, which makes our .hero-text element a child of the grid. We have access to a property for CSS Grid children called place-self. Calling place-self is a short-hand that sets both the horizontal and vertical alignment of the child item—not the elements inside the child, but the child itself. So we give that a value of center. This is the same as saying place-self: center center. It centers the element vertically centered (first value) and horizontally centered (second value) in its grid cell. You can learn more about place-self from this go-to resource at CSS-Tricks.

So now we've created easy centering with a fallback in less than 10 lines of code. It's not that difficult to get beautifully centered elements that you can also change using media queries for responsive design. You could play around with how this text reacts at different sizes.

Pro Tip: If your text isn't reacting the way if should with the Grid changes try commenting out the Flexbox code to see if that makes it work. Also, any styles applied to the child elements outside the feature query cascade down, so if you want an element to have different styles for Grid, you must override them in the @supports call.

Happy Centering!

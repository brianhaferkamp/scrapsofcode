Autofocus on Page Load

It's no secret that developing websites for mobile is important. There are some usability patterns that really help users when they visit your website. They're little touches that create delight for the user. The awesome part is that many of these moments of delight are really easy to implement.

One of the things I love is when I don't have to click to focus in on the first input of a form or signup page. This is a very nice touch for mobile users and requires only one word in the first input. You simply write autofocus as an attribute of the input you want to focus on page load. Here's a pen that shows what it's like when you have autofocus and without. You can see the attribute in the first form's email input.

See the Pen Focus on Page Load // Usability by Brian Haferkamp (@brianhaferkamp) CodePen.

You can see the forms in action by viewing the demo here. The two forms in the demo are exactly the same except for one word. Notice the top email address focuses on page load while the bottom input doesn't. To get the full effect, check it out on your mobile device.

This pattern is best for a page where the primary first step for the user is going to be the input. Consider it for login/signup pages, especially.

Designers Who Code, Coders Who Design

When I was first starting out there was one position for someone who built things on the Web: Webmaster. The webmaster designed and developed the website. Now we have front end and back end engineers, devOps, IS/IT, and designers.

I suppose I understand the disconnect between the front end and the back end. What I don't understand is separating design from front end development. I think the fact that there is often friction in the relationship of designers and front end engineers should tell us that those positions are not best separated.

To me, a front end developer should be able to design and a designer should be able to develop front end code--HTML, CSS, and a low-level knowledge of JavaScript.

In my experience a lot of friction happens between design and development because neither understands what the other is saying. I suppose one is predominantly right brained and the other left brained, but there doesn't seem to be any reason for them not to be able to speak the same language. Front end developers have often complained about the pickiness of designers and how things can't always be pixel perfect. Designers often complain about the limitations of the code and how their designs are changed or not accepted altogether.

If a designer knows the limitations of the code or how a web page or application is built then he or she could understand how their design would be implemented before he/she begins the design.

Likewise, if a developer understands how important the right colors, size, hierarchy, spacing, etc. are for communicating to the user then he or she could develop better code and understand why it's important to stick closely to the visual design.

My suggestion is that designers branch out and learn front end code (HTML, CSS, and JavaScript) and front end developers learn the basic concepts of design (spacing, hierarchy, typography, etc). Even if we have separation of our actual duties, being able to speak each other's languages could help to make better design and code choices.
